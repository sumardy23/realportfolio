Reflection

Some similarities are that Python and Scratch both have similar loops that you can use, and many of the features of Python and Scratch are the same, like the variables and what not. Some differences that I found were that Python was very specific about the ordering of the code, unlike Scratch, which was more lenient. Another difference was that the conditions of some of the loops and conditionals worked differently from language to language, and the different languages required different things.

For me, it felt like debugging took more time than actually writing the programme. Sometimes, it was kind of frustrating because once you thought that you fixed one part of the code, another problem would pop up just because of that one fix that you just made. Also, sometimes it was hard to find which part of the programme was causing the bug, as sometimes the errors were a result of an ordering mistake, which requires a lot of trial and error to fix.
